# ScenicPathfinder

个人系 2025 - 2026 学年数据结构课程课程设计「旅游区导览系统」

## 题目

### 题目描述

设计一个旅游区导航系统，能为访客提供信息查询功能和为管理员提供信息管理功能

### 基本要求

1. 设计某一旅游区的平面图，一般情况下，由于景点之间的道路是双向通行的，则设旅游区的平面图是一个无线网，其中图中顶点表示经典，存储的信息包括景点名称和景点简介等，要求景点数不少于 8 个；边表示景点之间的路径，存储的信息包括路径的长度和所需时间等，要求边数不少于 15 条
2. 可手动输入或随机生成某一旅游区的平面图信息，且图采用邻接数组或邻接表存储结构
3. 为管理员提供图中任意景点信息的编辑，包括添加、删除和修改景点信息
4. 为管理员提供图中任意景点之间路线信息的编辑，包括添加、删除和修改路线信息
5. 为访客提供途中任意景点信息的查询
6. 为访客提供任意两个景点之间路径信息的查询，即查询任意两个景点之间的最短路径

### 扩展要求

1. 为访客提供任意两个景点之间路径信息的查询，即查询任意两个景点之间的所有简单路径
2. 为访客规划一条能游览指定多个景点的简单路径信息
